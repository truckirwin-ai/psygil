================================================================================
                        ARCHITECTURE VERIFIED
================================================================================

IPC Protocol:
  - JSON-RPC 2.0 over Unix domain socket
  - Newline-delimited requests/responses
  - Auto-incrementing request IDs

Endpoints Tested:
  ✓ pii/detect — Entity detection (via test_server.py patterns)
  ✓ pii/redact — PHI → UNID conversion
  ✓ pii/rehydrate — UNID → PHI conversion
  ✓ pii/destroy — Explicit map destruction

UNID Format:
  - Pattern: {TYPE}_{6hex}
  - 24-bit entropy per UNID
  - Type prefixes: PERSON, DOB, DATE, ADDRESS, PHONE, EMAIL, SSN, etc.
  - No persistent storage (in-memory only)
  - Single-use per operation

Map Lifecycle:
  1. GENERATE — Fresh UNID map created for each operation
  2. REDACT — PHI replaced with UNIDs
  3. RECEIVE — AI response with UNIDs
  4. RE-HYDRATE — UNIDs replaced with original PHI
  5. DESTROY — Map zeroed from memory

Key Invariants Enforced:
  ✓ No PHI in any outbound API request
  ✓ No UNID map on disk
  ✓ No UNID reuse across operations
  ✓ Maps destroyed after rehydrate
  ✓ Full PHI available for reports (after local use)
