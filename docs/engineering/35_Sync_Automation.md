# Sync Automation (Phase G)

## Overview

Psygil Sync automates local git commits and pushes across three periodic triggers plus a pre-sleep hook. The system ensures that uncommitted work is never lost and that changes are synchronized to GitHub on a predictable schedule.

The sync system runs entirely in the background via macOS launchd agents. Installation is one-time; all agents are configured to run at startup.

## Installation

To install the sync system:

```bash
cd /Users/truckirwin/Desktop/Foundry\ SMB/Products/Psygil
bash scripts/setup-sync.sh install
```

The installer will:
- Copy four launchd agent plists to `~/Library/LaunchAgents/`
- Bootstrap each agent with launchctl
- Enable the `.githooks` directory via `git config core.hooksPath`
- Create `~/.psygil/` and initialize the sync log

Run `psygil-sync.sh status` to verify installation:

```bash
scripts/psygil-sync.sh status
```

## Sync Triggers

### 1. Periodic Commit (every 15 minutes)

Label: `com.foundrysmb.psygil.periodic`

If uncommitted tracked changes exist, creates a commit with the message `wip: auto-sync <timestamp>`.

Runs every 900 seconds (15 minutes) while the machine is active.

### 2. AFK Commit (every 5 minutes, if idle > 5 minutes)

Label: `com.foundrysmb.psygil.afk`

Checks every 300 seconds (5 minutes) whether the user has been idle for more than 300 seconds. If so, and if uncommitted changes exist, creates a commit with `wip: AFK auto-sync`.

Automatically detects idle time on macOS via the IOHIDSystem. Linux and Windows are not supported; the idle check returns 0 and the condition is skipped.

### 3. Hourly Push (every 60 minutes)

Label: `com.foundrysmb.psygil.hourly-push`

Pushes the current branch to origin if it is ahead of the remote. Runs every 3600 seconds (1 hour).

Branch protection prevents pushing to `main` or `master`; such pushes are logged and skipped.

### 4. Pre-Sleep Push (on power management changes)

Label: `com.foundrysmb.psygil.sleep-push`

Watches `/Library/Preferences/com.apple.powermanagement.plist` for changes, indicating the system is entering sleep. On detection, attempts to push any pending commits to origin.

Note: macOS power management file watching is not guaranteed to trigger on every sleep event. The hourly push trigger provides a fallback that runs at machine resume.

## Safety Gates

Every commit path applies the following safety gates:

### Hard Rule Scan

Staged changes are scanned for:
- Em dashes (U+2014)
- En dashes (U+2013)
- AI watermark strings ("Generated by AI", "Co-authored by Claude", "noreply@anthropic")

If any violations are detected, the commit is aborted and logged.

### Gitleaks Scan

If gitleaks is installed (`command -v gitleaks`), staged changes are scanned for potential secrets using `gitleaks protect --staged --verbose --redact`. If secrets are detected, the commit is aborted.

If gitleaks is not installed, the scan is skipped with a warning (does not block the commit).

### Branch Protection

Auto-push is never applied to `main` or `master`. Such attempts are logged and skipped.

### `.gitignore` Trust

Periodic and AFK commits use only `git add -u`, which stages only tracked files that are already in the index. This respects `.gitignore` and prevents accidental staging of new files.

Explicit `git add <path>` calls from Claude are still allowed during manual commits.

## Logging

All sync events are logged to `~/.psygil/sync.log` as JSON lines. Each entry includes:

```json
{
  "timestamp": "2026-04-16T14:30:00Z",
  "branch": "main",
  "sha": "a1b2c3d",
  "action": "commit",
  "result": "success",
  "message": "wip: auto-sync 2026-04-16T14:30:00Z"
}
```

View the last 5 events:

```bash
tail -5 ~/.psygil/sync.log | jq .
```

## CLI Commands

### commit

Stage tracked changes and create a commit.

```bash
scripts/psygil-sync.sh commit
scripts/psygil-sync.sh commit --message "manual save"
scripts/psygil-sync.sh commit --if-idle 300
```

Options:
- `--message "<msg>"`: Custom commit message (default: `wip: auto-sync <timestamp>`)
- `--if-idle <seconds>`: Skip commit unless idle time exceeds threshold (macOS only)

### push

Push the current branch to origin if ahead. Branch protection applies.

```bash
scripts/psygil-sync.sh push
```

### status

Print sync status, including the last commit SHA, idle seconds, uncommitted files, and recent log entries.

```bash
scripts/psygil-sync.sh status
```

### install-agents

Install launchd agents and git hooks (calls `setup-sync.sh install`).

```bash
scripts/psygil-sync.sh install-agents
```

### uninstall-agents

Uninstall launchd agents and restore git config (calls `setup-sync.sh uninstall`).

```bash
scripts/psygil-sync.sh uninstall-agents
```

## Managing Agents

### Pause All Agents

To temporarily disable all sync agents (e.g., during a rebase):

```bash
launchctl disable gui/$(id -u)/com.foundrysmb.psygil.periodic
launchctl disable gui/$(id -u)/com.foundrysmb.psygil.afk
launchctl disable gui/$(id -u)/com.foundrysmb.psygil.hourly-push
launchctl disable gui/$(id -u)/com.foundrysmb.psygil.sleep-push
```

### Resume All Agents

```bash
launchctl enable gui/$(id -u)/com.foundrysmb.psygil.periodic
launchctl enable gui/$(id -u)/com.foundrysmb.psygil.afk
launchctl enable gui/$(id -u)/com.foundrysmb.psygil.hourly-push
launchctl enable gui/$(id -u)/com.foundrysmb.psygil.sleep-push
```

### List Loaded Agents

```bash
launchctl list | grep psygil
```

## Uninstall

To remove the sync system:

```bash
scripts/psygil-sync.sh uninstall-agents
```

This will:
- Unload all four launchd agents
- Remove the agent plists from `~/Library/LaunchAgents/`
- Unset `git config core.hooksPath`

The log file (`~/.psygil/sync.log`) is preserved for historical reference.

## Locking

The sync system uses atomic `mkdir` for locking to prevent concurrent invocations. If an agent is already running, subsequent invocations exit silently (status code 0).

## Troubleshooting

### Agents Not Running

Check the log files:

```bash
tail -f /tmp/psygil-periodic.log
tail -f /tmp/psygil-periodic-err.log
```

List loaded agents:

```bash
launchctl list | grep psygil
```

Manually trigger a commit to test:

```bash
scripts/psygil-sync.sh commit
```

### Commits Not Being Created

Check for uncommitted changes:

```bash
git status
```

The sync system only commits tracked files. If you have new untracked files, add them manually:

```bash
git add path/to/new/file
```

Then wait for the next periodic trigger (15 minutes) or run:

```bash
scripts/psygil-sync.sh commit
```

### gitleaks Not Installed

The sync system skips gitleaks scans if the tool is not installed, with a warning logged. To enable gitleaks scanning:

```bash
brew install gitleaks
```

Alternatively, install via the official release:

```bash
# https://github.com/gitleaks/gitleaks/releases
```

### Pre-Commit Hook Failures

If `git commit` fails due to HARD RULE violations, fix the staged changes and try again:

```bash
git diff --cached | less  # Review staged changes
# Fix violations in the source files
git add -u  # Re-stage fixed changes
git commit -m "your message"
```

## Design Notes

The sync system prioritizes safety over automation:

- Every commit path runs safety gates (HARD RULE scan, gitleaks scan)
- Branch protection prevents accidental pushes to main/master
- Locking prevents concurrent invocations from interfering
- Logging captures every action for auditing and debugging
- No automatic pushes to protected branches
- macOS idle detection is macOS-specific; other platforms are supported but without idle detection

The system is designed for solo development where uncommitted work loss is the primary concern. In team environments with shared branches, branch protection rules on GitHub provide additional safeguards.
